# Тестовое задание (mutex map + тест)

## Описание задачи

Реализовать `map`, защищённый мьютексом, и написать для него тест.

Условия:

- Ключи в мапе должны быть **числовыми**.
- Количество ключей равно **году рождения известной личности** или **году известного события**.
  - В комментарии к коду нужно указать источник года.
  - Диапазон ключей: **от 1 до выбранного года**.
- Мапу заранее ключами и значениями **заполнять нельзя**.
- В тесте нужно запустить **четыре горутины**.

Алгоритм работы горутин:

- Обращение к значению по ключу. Если значения нет — оно должно быть **создано**.
- Увеличение значения на 1.

Важно:

- У мапы не должно быть **отдельной функции инкремента** значения.

Ожидаемый результат:

- После завершения всех горутин, для каждого ключа должно лежать число **3**.
- Горутины не должны обращаться к ключам последовательно (ни по возрастанию, ни по убыванию).
- В мапу нужно добавить:
  - счётчик обращений к ключам
  - счётчик добавлений новых ключей
- В конце теста:
  - счётчик обращений = год * 3
  - счётчик добавлений = год
- В комментарии к коду указать **ФИО автора**.

## Реализация

- Пакет: `safemap`
- Основной код: `safemap/safemap.go`
  - `SafeMap` содержит `map[int]int`, `sync.Mutex`, а также счётчики `accesses` и `adds`.
  - Метод `WithValue(key, fn)` выполняет переданную функцию под мьютексом: создаёт значение при отсутствии и позволяет увеличить значение (без отдельного `Increment`-метода).
- Тест: `safemap/safemap_test.go`
  - Генерируется список операций длиной `year*3` (каждый ключ встречается 3 раза), перемешивается и делится между 4 горутинами.
  - Проверяется, что итоговые значения и счётчики соответствуют условиям.

## Выбранный год

В коде выбран год **79** (извержение Везувия и гибель Помпей, 79 год н.э.). Источник указан в комментарии в `safemap/safemap.go`.
